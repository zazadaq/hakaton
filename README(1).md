# Прогноз дохода клиентов банка — гайд по запуску

Короткая инструкция: что запускать, как запускать и что настраивать.

---

## 1. С чего начать (кратко)

1. Установите Python и библиотеки (раздел 2).
2. Положите `train.csv` и `test.csv` в папку с блокнотом.
3. Откройте **`income_final_commented.ipynb`** и выполните все ячейки сверху вниз.
4. Заберите готовый файл **`commit.csv`** — это ваш сабмит.

Лучший результат на лидерборде: **75 938**.

---

## 2. Что установить

Нужен **Python 3.10–3.11**. Проверить: в терминале `python --version`.

Установка библиотек одной командой:
```bash
pip install pandas numpy lightgbm scikit-learn scipy jupyter
```

Требования к железу:
- **Процессор** — считает на CPU (видеокарта не нужна и не ускорит). Больше ядер = быстрее.
- **Память** — 2–4 ГБ достаточно.
- **Время полного прогона** — ~7–12 минут на мощном CPU (напр. Ryzen 7 5700X3D), ~25–40 минут на слабом/облачном.

---

## 3. Как запустить

### Вариант А — Jupyter (браузер)
```bash
cd путь/к/папке-с-блокнотом
jupyter notebook
```
Откроется вкладка в браузере → кликните на блокнот → меню **Kernel → Restart & Run All** → дождитесь конца.

### Вариант Б — VS Code
Откройте `.ipynb` в VS Code (он предложит поставить поддержку Jupyter) → кнопка **Run All**.

### Вариант В — без браузера, одной командой
```bash
jupyter nbconvert --to notebook --execute income_final_commented.ipynb
```
Прогонит все ячейки и создаст `commit.csv`.

> **Важно:** ячейки зависят друг от друга и должны выполняться по порядку. Всегда используйте **Run All**, а не запуск одной последней ячейки — иначе будет ошибка вида `name 'TAIL_T' is not defined` (переменные задаются в ячейках выше). Ждите, пока слева у ячейки появится число `[5]`, а не звёздочка `[*]` (звёздочка = ещё считается).

## 4. Результат

После прогона в папке появляется **`commit.csv`** — таблица `id;predict` (десятичная запятая). Это файл для отправки на лидерборд.

Готовый заранее посчитанный сабмит (если не хотите ждать прогон): **`commit_best.csv`** — финальные настройки (`TAIL_T=80000, TAIL_A=0.7, A_SAL=0.65`), уровень 75 938.

---

## 5. Что можно настраивать (и как быстро видеть эффект)

Все «крутилки» — в **последней ячейке** блокнота. Тяжёлые ячейки (обучение) их не используют, поэтому:

> Прогоните блокнот один раз целиком. Дальше меняйте параметры **только в последней ячейке** и жмите на ней **Shift+Enter** — пересчёт занимает секунды (модели уже обучены), новый `commit.csv` готов сразу.

| Параметр | Что делает | Текущее | Куда крутить |
|---|---|---|---|
| `TAIL_A` | сила подъёма прогноза для богатых | `0.7` | 0.6–0.75, главный рычаг |
| `TAIL_T` | порог, с которого поднимаем | `80000` | 60000–100000 |
| `A_SAL` | доверие прямой зарплате | `0.65` | 0.6–0.85 |
| `SEEDS` | сколько запусков усредняем (устойчивость) | `[42, 2024]` | `[42]` быстрее; `[42,2024,7]` стабильнее |

Число фолдов — в ячейке финального обучения: `KFold(n_splits=5, ...)`. 5 достаточно; больше почти не даёт прироста.

> Абсолютную ошибку локально вы не увидите — её знает только лидерборд. Чтобы сравнивать варианты честно, смотрите WMAE в ячейке валидации (honest holdout) или отправляйте несколько файлов на лидерборд.

---

## 6. Как устроено решение (в двух словах)

Метрика взвешена так, что почти вся ошибка — на богатых клиентах. Поэтому:
1. **Признаки** — агрегаты по прокси дохода + кодирование категорий.
2. **Ансамбль** — две LightGBM (простая + глубокая) усредняются; третья, квантильная, — для хвоста.
3. **Коррекция хвоста** — точечно поднимаем высокие прогнозы.
4. **Override зарплатой** — где известна прямая зарплата (корр. 0.935), доверяем ей.

Путь качества на валидации: 34 177 → 33 895 → 33 680 → 33 476 → 32 944. На лидерборде: 77 184 → 75 938.
